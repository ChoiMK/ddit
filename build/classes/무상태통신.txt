Stateless(무상태) 통신방식 : http 프로토콜
1. 장,단점
   1.1 장점 : 작은 메모리 활용
                경량화된 스레드를 활용
                로드밸렁싱 가능
                서버 확장이 용이

   1.2 단점 : 요청시 헤더정보 사이즈가 statful(TCP Socket 통신)보다 큼                
             statful 보다 느림
             단방향 통신
             
2. 특징
    2.1 클라이언트와 웹서버와의 요청과 응답 통신은 statelss를 지향함.
          클라이언트가 서버로 요청 전송 후에 통신 접속은 단절,
          서버가 클라이언트에 응답 전송 후에 통신 접속은 단절.         
    2.2 클라이언트와 웹서버 요청, 응답시 지속적으로 유지되어야하는 정보가 존재한다면
          클라이언트 측은 Cookie, 서버 측은 session을 활용.

3. 활용
    3.1 쇼핑몰에서 전날에 내가 열람한 상품 정보가 금일 해당 쇼핑몰 접근시 출력.
    3.2 회원 로그인 후 회원 권한으로 특정 사이트를 서비스 이용.
    3.3 특정 사이트에서 팝업 출력시 '오늘은 더이상 보지 않음 체크' 팝업 미출력.
